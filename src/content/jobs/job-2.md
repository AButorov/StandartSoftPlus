## Задача бизнеса

Региональный перевозчик столкнулся с необходимостью конкуренции с федеральными агрегаторами (Yandex). Существующая модель работы через телефонную диспетчерскую ограничивала рост: долгое время обработки заказа, сложность контроля водителей на линии и отсутствие прозрачности для пассажиров.

**Цели разработки:**

- Создать собственную экосистему (аналог Uber/Yandex) для удержания доли локального рынка
- Автоматизировать распределение заказов между водителями для снижения нагрузки на диспетчеров
- Внедрить гибридную систему ценообразования (автоматический расчет + контроль диспетчера)

## Реализованная архитектура

Команда StandartSoft разработала комплексное решение, объединяющее три модуля в единую облачную систему.

### Мобильное приложение для пассажиров

Интуитивный интерфейс для быстрого вызова автомобиля без звонка оператору:

- **Геолокация** — автоматическое определение места подачи и выбор конечной точки на карте
- **Информирование** — отображение данных о водителе (фото, рейтинг) и автомобиле (марка, цвет, госномер)
- **Прозрачность** — отслеживание приближения автомобиля в реальном времени и push-уведомления о статусах заказа
- **Обратная связь** — система оценки поездок и отзывов для контроля качества сервиса

### Мобильное приложение для водителей

Рабочее место водителя, автоматизирующее рутинные процессы:

- **Управление сменой** — фотоконтроль документов и автомобиля при выходе на линию
- **Работа с заказами** — прием заказов с таймером на принятие решения, встроенный таксометр для расчета стоимости ожидания клиента
- **Баланс и история** — просмотр статистики выполненных поездок и заработка

### Веб-панель диспетчера

Центр управления операционной деятельностью таксопарка:

- **Мониторинг** — интерактивная карта с отображением всех активных бортов в реальном времени (GPS-трекинг)
- **Диспетчеризация** — возможность ручного создания заказов (по телефону) и валидация стоимости для сложных маршрутов
- **Управление парком** — верификация водителей и автомобилей, блокировка нарушителей, назначение машин из парка предприятия

## Технические особенности

Для обеспечения справедливости распределения заказов и минимизации времени подачи был реализован кастомный алгоритм **«Очередь + Радиус»**:

1. **Динамический поиск** — система ищет ближайшего водителя в заданном радиусе
2. **Расширение зоны** — если водитель не найден, радиус поиска автоматически увеличивается
3. **Приоритет очереди** — при наличии нескольких кандидатов заказ предлагается тому, кто дольше находится в очереди ожидания

**Стек технологий:**

- **Backend:** Облачная инфраструктура обеспечивает отказоустойчивость 24/7
- **Real-time DB:** Firebase для мгновенной синхронизации статусов заказов между диспетчером, водителем и клиентом
- **Картография:** Интеграция с картографическими сервисами для точного построения маршрутов и геокодинга адресов

## Результат

Проект реализован за 4 месяца в полном соответствии с техническим заданием. Внедрение комплекса позволило заказчику:

- Сократить время подачи автомобиля за счет автоматического алгоритма распределения
- Разгрузить диспетчерскую службу, переведя часть потока заказов в приложение
- Повысить лояльность клиентов и водителей за счет прозрачных условий работы
