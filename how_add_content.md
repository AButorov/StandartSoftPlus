## Где находятся проекты

Все проекты хранятся в папке:

```
src/content/jobs/
```

Каждый проект — это отдельный файл `.md` (например, `job-1.md`, `job-2.md`)

**Изображения проектов хранятся в:**

```
public/images/projects/
```

## Добавление нового проекта

### Шаг 1: Создайте файл

Создайте новый файл в папке `src/content/jobs/`:

```
src/content/jobs/job-3.md
```

### Шаг 2: Скопируйте шаблон

```markdown
---
title: "Название проекта"
description: "Краткое описание (одна строка для карточки)"
details: "Развёрнутое описание для карточки на странице портфолио"
tags: ["Тег1", "Тег2", "Тег3"]
client: "Название клиента"
stack: "Технологии через запятую"
status: "Сдан"
features:
  - "Основная возможность 1"
  - "Основная возможность 2"
  - "Основная возможность 3"
  - "Основная возможность 4"
order: 3
pubDate: "2025-06-01"
---

## Заголовок раздела

Полное описание проекта. Здесь можно использовать обычный Markdown:

- Списки
- **Жирный текст**
- Параграфы

### Подзаголовок

Продолжение описания...

---

**Клиент:** Название компании  
**Стек:** Технологии  
**Статус:** Статус проекта
```

### Шаг 3: Заполните данные

**Обязательные поля в шапке (между `---`):**

| Поле          | Описание             | Пример                                           |
| ------------- | -------------------- | ------------------------------------------------ |
| `title`       | Название проекта     | `'Мой новый проект'`                             |
| `description` | Короткое описание    | `'Сайт для продажи товаров'`                     |
| `details`     | Развёрнутое описание | `'Полный интернет-магазин с корзиной и оплатой'` |
| `tags`        | Теги (массив)        | `['Web', 'E-commerce']`                          |
| `client`      | Клиент               | `'ООО "Компания"'`                               |
| `stack`       | Технологии           | `'React, Node.js, PostgreSQL'`                   |
| `status`      | Статус               | `'Сдан'` / `'В работе'`                          |
| `features`    | Возможности (массив) | См. шаблон                                       |
| `order`       | Порядок сортировки   | `3` (число)                                      |
| `pubDate`     | Дата (опционально)   | `'2025-06-01'`                                   |

**⚠️ Важно:**

- Текст в одинарных кавычках: `'текст'`
- Списки с дефисом и пробелом: `- 'элемент'`
- `order` — число без кавычек (определяет порядок на странице портфолио)

### Шаг 4: Перезапустите сервер

```bash
# Остановите сервер (Ctrl+C)
# Запустите заново
bun run dev
```

Проект появится автоматически:

- В списке: `http://localhost:4321/portfolio/`
- Страница проекта: `http://localhost:4321/jobs/job-3/`

## Редактирование проекта

### Изменить данные в карточке

Откройте файл проекта (например, `src/content/jobs/job-1.md`) и измените нужные поля в шапке:

```markdown
---
title: "Новое название"
description: "Новое описание"
stack: "Новые технологии"
---
```

Сохраните файл — изменения применятся автоматически.

### Изменить содержание страницы

Измените текст **после** второй строки `---`:

```markdown
---
title: "Проект"
...

---

## Это содержание страницы

Здесь пишите что угодно в формате Markdown.
```

## Работа с изображениями

### Структура папок для изображений

Создайте следующую структуру:

```
public/
  images/
    projects/
      job-1/
        hero.jpg
        screenshot-1.jpg
        screenshot-2.jpg
      job-2/
        hero.jpg
        demo.jpg
      job-3/
        ...
```

**Принцип:** для каждого проекта — своя папка с изображениями.

### Добавление изображений

**Шаг 1: Создайте папку для проекта**

```bash
mkdir -p public/images/projects/job-3
```

**Шаг 2: Скопируйте изображения**

Поместите файлы в папку:

```
public/images/projects/job-3/
  ├── hero.jpg           ← главное изображение
  ├── interface.png      ← скриншот интерфейса
  └── architecture.png   ← схема архитектуры
```

**Шаг 3: Вставьте в описание проекта**

Откройте файл проекта `src/content/jobs/job-3.md`:

```markdown
---
title: "Мой проект"
...

---

## Описание

Здесь текст описания.

![Главный экран приложения](/images/projects/job-3/hero.jpg)

Продолжение текста...

### Интерфейс

![Скриншот интерфейса](/images/projects/job-3/interface.png)

### Архитектура

![Схема архитектуры](/images/projects/job-3/architecture.png)
```

### Синтаксис вставки изображений

**Базовый синтаксис:**

```markdown
![Описание изображения](/images/projects/job-X/image.jpg)
```

- `![...]` — маркер изображения
- `Описание изображения` — alt-текст (для SEO и доступности)
- `/images/projects/job-X/image.jpg` — путь к файлу

**С подписью:**

```markdown
![Главный экран](/images/projects/job-3/hero.jpg)
_Рисунок 1: Главный экран приложения_
```

**Несколько изображений подряд:**

```markdown
![Экран 1](/images/projects/job-3/screen-1.jpg)
![Экран 2](/images/projects/job-3/screen-2.jpg)
![Экран 3](/images/projects/job-3/screen-3.jpg)
```

### Рекомендации по изображениям

**Форматы файлов:**

- **Фотографии / скриншоты:** `.jpg` или `.webp`
- **Схемы / иконки:** `.png` или `.svg`
- **Анимация:** `.gif` (использовать редко)

**Размеры:**

- **Ширина:** максимум 1200px (для быстрой загрузки)
- **Вес:** до 500KB на изображение
- **Hero-изображение:** 1200x600px (пропорция 2:1)

**Названия файлов:**

```
✅ Хорошо:
- hero.jpg
- interface-main.jpg
- architecture-diagram.png

❌ Плохо:
- IMG_1234.jpg
- Снимок экрана 2025-01-05.png
- image (1).jpg
```

### Оптимизация изображений

**Сжатие онлайн:**

- [TinyPNG](https://tinypng.com/) — для PNG/JPG
- [Squoosh](https://squoosh.app/) — универсальный

**Через командную строку (опционально):**

Установка ImageMagick:

```bash
# macOS
brew install imagemagick

# Ubuntu/Debian
sudo apt install imagemagick
```

Сжатие изображения:

```bash
# Изменить размер до 1200px ширины
convert input.jpg -resize 1200x output.jpg

# Сжать с качеством 85%
convert input.jpg -quality 85 output.jpg
```

### Удаление изображений

Просто удалите файлы из папки проекта:

```bash
rm public/images/projects/job-3/old-screenshot.jpg
```

И удалите ссылку из `.md` файла.

### Переименование изображения

1. Переименуйте файл:

```bash
mv public/images/projects/job-3/old.jpg public/images/projects/job-3/new.jpg
```

2. Обновите ссылку в `.md` файле:

```markdown
![Описание](/images/projects/job-3/new.jpg)
```

### Если изображение не отображается

Проверьте:

1. **Путь правильный?** Должен начинаться с `/images/projects/`
2. **Файл существует?** Проверьте папку `public/images/projects/job-X/`
3. **Расширение правильное?** `.jpg` vs `.JPG` (регистр важен на Linux)
4. **Перезапустили сервер?** `bun run dev`

## Расширенное размещение изображений

Стандартный Markdown **не ограничивает** возможности размещения изображений. Можно использовать HTML внутри `.md` файлов для точного контроля.

### Выравнивание изображений

#### По центру (базовый Markdown)

```markdown
![Описание](/images/projects/job-3/hero.jpg)
```

Изображение автоматически будет по центру на всю ширину контейнера.

#### По центру (с CSS классом)

```html
<img src="/images/projects/job-3/hero.jpg" alt="Описание" class="img-center" />
```

#### По левому краю с обтеканием текстом

```html
<img
  src="/images/projects/job-3/interface.jpg"
  alt="Интерфейс"
  class="img-left"
/>

Этот текст будет обтекать изображение справа. Изображение займёт 50% ширины
контейнера и будет прижато к левому краю. Отлично подходит для демонстрации
небольших скриншотов или иконок с пояснительным текстом.

<div class="clear"></div>
<!-- Очистка обтекания -->
```

#### По правому краю с обтеканием текстом

```html
<img src="/images/projects/job-3/logo.png" alt="Логотип" class="img-right" />

Этот текст будет обтекать изображение слева. Изображение займёт 50% ширины
контейнера и будет прижато к правому краю. Используйте для логотипов, портретов
или акцентных изображений.

<div class="clear"></div>
<!-- Очистка обтекания -->
```

### Два изображения рядом

#### Вариант 1: Flex контейнер (универсальный)

```html
<div class="img-row">
  <img src="/images/projects/job-3/screen-1.jpg" alt="Экран 1" />
  <img src="/images/projects/job-3/screen-2.jpg" alt="Экран 2" />
</div>
```

**Особенности:**

- Изображения занимают равное место
- Адаптивно (на мобильных переходят в колонку)
- Работает с любым количеством изображений

#### Вариант 2: Grid для двух изображений

```html
<div class="img-row-2">
  <img src="/images/projects/job-3/before.jpg" alt="До" />
  <img src="/images/projects/job-3/after.jpg" alt="После" />
</div>
```

**Особенности:**

- Строго два изображения рядом
- Чёткая сетка 50/50
- На мобильных — вертикально

### Три изображения рядом

```html
<div class="img-row-3">
  <img src="/images/projects/job-3/step-1.jpg" alt="Шаг 1" />
  <img src="/images/projects/job-3/step-2.jpg" alt="Шаг 2" />
  <img src="/images/projects/job-3/step-3.jpg" alt="Шаг 3" />
</div>
```

**Особенности:**

- Три изображения в ряд
- На планшетах и мобильных — вертикально

### Смешанные макеты

```html
## Интерфейс приложения Мы создали интуитивный интерфейс с фокусом на простоту
использования.

<img
  src="/images/projects/job-3/main-screen.jpg"
  alt="Главный экран"
  class="img-center"
/>

### Детали интерфейса

<div class="img-row-2">
  <img src="/images/projects/job-3/menu.jpg" alt="Меню" />
  <img src="/images/projects/job-3/settings.jpg" alt="Настройки" />
</div>

<img
  src="/images/projects/job-3/feature-icon.png"
  alt="Иконка функции"
  class="img-left"
/>

Основная функция приложения позволяет пользователям быстро выполнять задачи.
Интерфейс разработан с учётом лучших практик UX/UI дизайна.

<div class="clear"></div>

### Этапы работы

<div class="img-row-3">
  <img src="/images/projects/job-3/stage-1.jpg" alt="Этап 1" />
  <img src="/images/projects/job-3/stage-2.jpg" alt="Этап 2" />
  <img src="/images/projects/job-3/stage-3.jpg" alt="Этап 3" />
</div>
```

### Доступные CSS классы

| Класс        | Описание                                 | Использование                |
| ------------ | ---------------------------------------- | ---------------------------- |
| `img-center` | Выравнивание по центру                   | Основные изображения         |
| `img-left`   | Выравнивание слева с обтеканием текстом  | Иконки, логотипы, портреты   |
| `img-right`  | Выравнивание справа с обтеканием текстом | Акцентные изображения        |
| `img-row`    | Flex контейнер для изображений рядом     | Любое количество изображений |
| `img-row-2`  | Grid для двух изображений                | "До/После", сравнения        |
| `img-row-3`  | Grid для трёх изображений                | Этапы, категории             |
| `clear`      | Очистка обтекания                        | После img-left / img-right   |

### Адаптивность

Все классы автоматически адаптируются под мобильные устройства:

- **Десктоп (> 768px):** изображения рядом, float работает
- **Мобильный (≤ 768px):** изображения вертикально, float отключён

### Когда использовать HTML вместо Markdown

| Ситуация                         | Синтаксис         |
| -------------------------------- | ----------------- |
| Одно изображение на всю ширину   | Markdown `![...]` |
| Выравнивание (left/right/center) | HTML + класс      |
| Два и более изображения рядом    | HTML + класс      |
| Обтекание текстом                | HTML + класс      |
| Точный контроль размеров         | HTML + style      |

## Изменение порядка проектов

Проекты сортируются по полю `order` (от меньшего к большему).

**Пример:**

```markdown
# job-1.md

order: 1 ← будет первым

# job-2.md

order: 2 ← будет вторым

# job-3.md

order: 3 ← будет третьим
```

Чтобы поменять местами проекты 1 и 2:

```markdown
# job-1.md

order: 2 ← теперь второй

# job-2.md

order: 1 ← теперь первый
```

## Частые ошибки

### ❌ Не работает — забыли кавычки

```markdown
title: Проект без кавычек ← ОШИБКА
```

**✅ Правильно:**

```markdown
title: 'Проект в кавычках'
```

---

### ❌ Не работает — неправильный список

```markdown
features:

- Без кавычек ← ОШИБКА
```

**✅ Правильно:**

```markdown
features:

- 'С кавычками'
- 'И отступом (2 пробела)'
```

---

### ❌ Не работает — order в кавычках

```markdown
order: '3' ← ОШИБКА (это число!)
```

**✅ Правильно:**

```markdown
order: 3
```

---

### ❌ Изображение не отображается

```markdown
![Описание](images/projects/job-3/hero.jpg) ← ОШИБКА (нет /)
```

**✅ Правильно:**

```markdown
![Описание](/images/projects/job-3/hero.jpg)
```

## Техническая реализация

![Архитектура приложения](/images/projects/job-3/architecture.png)

Интеграция с Apple HealthKit для автоматического импорта данных о тренировках и сне.

**Стек технологий:**

- **Frontend:** Swift, SwiftUI
- **Backend:** Firebase (Firestore, Cloud Functions)
- **Интеграции:** HealthKit, Push Notifications

## Проверка

После изменений проверьте:

1. **Локально:** `bun run dev` → откройте `http://localhost:4321/portfolio/`
2. **Ошибки:** смотрите в терминал — там будут сообщения об ошибках
3. **Изображения:** все ли картинки отображаются корректно
4. **Выравнивание:** проверьте классы выравнивания
5. **Адаптивность:** откройте DevTools и проверьте на мобильных разрешениях
6. **Мобильная версия:** проверьте как выглядит на узком экране

## Вопросы?

Если что-то не работает:

1. Проверьте кавычки в шапке
2. Проверьте отступы у списков (2 пробела)
3. Проверьте `order` (должно быть число)
4. Проверьте пути к изображениям (начинаются с `/`)
5. Проверьте закрытие тегов HTML (`<div class="clear"></div>`)
6. Перезапустите сервер: `bun run dev`
7. Посмотрите ошибки в терминале
