# Инструкция по работе с проектами

## Где находятся проекты

Все проекты хранятся в папке:

```
src/content/jobs/
```

Каждый проект — это отдельный файл `.md` (например, `job-1.md`, `job-2.md`)

---

## Добавление нового проекта

### Шаг 1: Создайте файл

Создайте новый файл в папке `src/content/jobs/`:

```
src/content/jobs/job-3.md
```

### Шаг 2: Скопируйте шаблон

```markdown
---
title: "Название проекта"
description: "Краткое описание (одна строка для карточки)"
details: "Развёрнутое описание для карточки на странице портфолио"
tags: ["Тег1", "Тег2", "Тег3"]
client: "Название клиента"
stack: "Технологии через запятую"
status: "Сдан"
features:
  - "Основная возможность 1"
  - "Основная возможность 2"
  - "Основная возможность 3"
  - "Основная возможность 4"
order: 3
pubDate: "2025-06-01"
---

## Заголовок раздела

Полное описание проекта. Здесь можно использовать обычный Markdown:

- Списки
- **Жирный текст**
- Параграфы

### Подзаголовок

Продолжение описания...

---

**Клиент:** Название компании  
**Стек:** Технологии  
**Статус:** Статус проекта
```

### Шаг 3: Заполните данные

**Обязательные поля в шапке (между `---`):**

| Поле          | Описание             | Пример                                           |
| ------------- | -------------------- | ------------------------------------------------ |
| `title`       | Название проекта     | `'Мой новый проект'`                             |
| `description` | Короткое описание    | `'Сайт для продажи товаров'`                     |
| `details`     | Развёрнутое описание | `'Полный интернет-магазин с корзиной и оплатой'` |
| `tags`        | Теги (массив)        | `['Web', 'E-commerce']`                          |
| `client`      | Клиент               | `'ООО "Компания"'`                               |
| `stack`       | Технологии           | `'React, Node.js, PostgreSQL'`                   |
| `status`      | Статус               | `'Сдан'` / `'В работе'`                          |
| `features`    | Возможности (массив) | См. шаблон                                       |
| `order`       | Порядок сортировки   | `3` (число)                                      |
| `pubDate`     | Дата (опционально)   | `'2025-06-01'`                                   |

**⚠️ Важно:**

- Текст в одинарных кавычках: `'текст'`
- Списки с дефисом и пробелом: `- 'элемент'`
- `order` — число без кавычек (определяет порядок на странице портфолио)

### Шаг 4: Перезапустите сервер

```bash
# Остановите сервер (Ctrl+C)
# Запустите заново
bun run dev
```

Проект появится автоматически:

- В списке: `http://localhost:4321/portfolio/`
- Страница проекта: `http://localhost:4321/jobs/job-3/`

## Редактирование проекта

### Изменить данные в карточке

Откройте файл проекта (например, `src/content/jobs/job-1.md`) и измените нужные поля в шапке:

```markdown
---
title: "Новое название"
description: "Новое описание"
stack: "Новые технологии"
---
```

Сохраните файл — изменения применятся автоматически.

### Изменить содержание страницы

Измените текст **после** второй строки `---`:

```markdown
---
title: "Проект"
...

---

## Это содержание страницы

Здесь пишите что угодно в формате Markdown.
```

---

## Изменение порядка проектов

Проекты сортируются по полю `order` (от меньшего к большему).

**Пример:**

```markdown
# job-1.md

order: 1 ← будет первым

# job-2.md

order: 2 ← будет вторым

# job-3.md

order: 3 ← будет третьим
```

Чтобы поменять местами проекты 1 и 2:

```markdown
# job-1.md

order: 2 ← теперь второй

# job-2.md

order: 1 ← теперь первый
```

## Удаление проекта

Просто удалите файл:

```bash
rm src/content/jobs/job-3.md
```

Перезапустите сервер:

```bash
bun run dev
```

Проект исчезнет из портфолио.

## Форматирование текста (Markdown)

Внутри описания проекта (после `---`) можно использовать:

### Заголовки

```markdown
## Большой заголовок

### Средний заголовок

#### Маленький заголовок
```

### Списки

```markdown
- Пункт 1
- Пункт 2
- Пункт 3

1. Нумерованный 1
2. Нумерованный 2
```

### Выделение

```markdown
**Жирный текст**
_Курсив_
```

### Разделитель

```markdown
---
```

### Ссылки

```markdown
[Текст ссылки](https://example.com)
```

## Частые ошибки

### ❌ Не работает — забыли кавычки

```markdown
title: Проект без кавычек ← ОШИБКА
```

### ✅ Правильно

```markdown
title: 'Проект в кавычках'
```

---

### ❌ Не работает — неправильный список

```markdown
features:

- Без кавычек ← ОШИБКА
```

### ✅ Правильно

```markdown
features:

- 'С кавычками'
- 'И отступом (2 пробела)'
```

---

### ❌ Не работает — order в кавычках

```markdown
order: '3' ← ОШИБКА (это число!)
```

### ✅ Правильно

```markdown
order: 3
```

## Полный пример нового проекта

**Файл:** `src/content/jobs/job-3.md`

```markdown
---
title: "Интернет-магазин одежды"
description: "Онлайн-платформа для продажи брендовой одежды"
details: "Полнофункциональный интернет-магазин: каталог товаров, корзина, онлайн-оплата, личный кабинет"
tags: ["E-commerce", "Web", "Mobile"]
client: 'ООО "Модная одежда"'
stack: "React, Node.js, PostgreSQL, Stripe"
status: "Сдан"
features:
  - "Каталог с фильтрами"
  - "Корзина и оформление заказа"
  - "Онлайн-оплата картой"
  - "Личный кабинет покупателя"
order: 3
pubDate: "2025-07-15"
---

## Задача бизнеса

Клиент хотел запустить онлайн-продажи брендовой одежды с удобным интерфейсом и быстрой оплатой.

**Ключевые требования:**

- Простой и понятный каталог
- Быстрое оформление заказа
- Интеграция с платёжными системами

## Реализованное решение

Мы создали современный интернет-магазин с адаптивным дизайном для всех устройств.

### Основные возможности

- **Каталог товаров** — удобные фильтры по размеру, цвету, цене
- **Корзина** — быстрое добавление и оформление
- **Оплата** — интеграция со Stripe для безопасных платежей
- **Личный кабинет** — история заказов и избранное

## Результат

Проект запущен за 3 месяца. Средний чек вырос на 25%, конверсия увеличилась на 40%.

---

**Клиент:** ООО "Модная одежда" (г. Москва)  
**Стек:** React, Node.js, PostgreSQL, Stripe  
**Статус:** Проект сдан и работает в production
```

## Вопросы?

Если что-то не работает:

1. Проверьте кавычки в шапке
2. Проверьте отступы у списков (2 пробела)
3. Проверьте `order` (должно быть число)
4. Перезапустите сервер: `bun run dev`
